# lab13
Лабораторная работа №13 — Система непересекающихся множеств и минимальное остовное дерево

Представьте: у вас N зданий в городе и нужно протянуть оптимальную сеть кабелей, чтобы соединить все здания с минимальной суммарной длиной. Классическая задача — минимальное остовное дерево (MST). А для её эффективного решения алгоритмом Краскала потребуется специальная структура данных — Система непересекающихся множеств (СНМ, или DSU — Disjoint Set Union).

СНМ отвечает на один вопрос молниеносно: «принадлежат ли два элемента одному компоненту связности?». Именно это нужно Краскалу на каждом шаге — чтобы не допустить цикл в остовном дереве.

---

### Цель работы

Построить минимальную сеть для **города, где вы родились** — соединить все здания кабелем с минимальной суммарной длиной.

## Задача 1: Система непересекающихся множеств (DSU)

СНМ поддерживает три операции:
- **MakeSet(x)** — создать одноэлементное множество `{x}`
- **FindSet(x)** — найти представителя множества, которому принадлежит `x`
- **Union(x, y)** — объединить множества, содержащие `x` и `y`

Пример применения: определить, в одном ли компоненте связности находятся два узла графа.

### Постановка задачи

Реализовать DSU поверх `HashTable` из lab4 **любым способом**. Ключ — `long` (идентификатор узла), значение — указатель на `DSNode`.

### Требования к реализации

```c
typedef struct DSNode {
    long value;
    struct DSNode *parent;
} DSNode;

typedef struct {
    HashTable *trees;  // long -> DSNode*
} DisjointSet;

// Создание узла (MakeSet)
DSNode *make_node(long value);

// Создание DSU
DisjointSet *dsu_create();
DisjointSet *dsu_create_from_hash_table(HashTable *nodes);

// FindSet — поиск представителя
DSNode *find_set(DSNode *node);
DSNode *find_set_from_key(DisjointSet *dsu, long key);

// Union — объединение множеств
void union_sets(DSNode *a, DSNode *b);

void freeDisjointSet(DisjointSet *dsu);
```

### Пример работы

```
MakeSet: {1}, {2}, {3}, {4}, {5}

Union(1, 2): {1, 2}, {3}, {4}, {5}
Union(3, 4): {1, 2}, {3, 4}, {5}
Union(2, 3): {1, 2, 3, 4}, {5}

FindSet(4) == FindSet(1)?  → ДА  (один компонент)
FindSet(4) == FindSet(5)?  → НЕТ (разные компоненты)
```

### Требования к тестам

Тесты размещаются в `test_disjoint_set.c`. Проверить:

1. `make_node` — узел создан, `parent == node`
2. `dsu_create` — пустая структура создана корректно
3. `dsu_create_from_hash_table` — каждый ключ доступен через `find_set_from_key`
4. `find_set` — одиночный узел является сам себе представителем
5. `union_sets` — после объединения `find_set(a) == find_set(b)`
6. Транзитивность: `union(a,b)`, `union(b,c)` → `find_set(a) == find_set(c)`
7. Идемпотентность: повторный `union` не ломает структуру

## Задача 2: Алгоритм Краскала + визуализация на карте

Используем DSU для построения минимального остовного дерева на реальном графе зданий города.

### Постановка задачи

Дан граф, где узлы — здания (с координатами lat/lon), рёбра — расстояния между ними. Найти минимальное остовное дерево (MST) — подграф, связывающий все узлы с минимальной суммарной длиной рёбер.

### Алгоритм Краскала

1. Сортируем все рёбра по возрастанию веса
2. Создаём DSU, где каждый узел — отдельное множество
3. Перебираем рёбра в порядке возрастания:
   - Если концы ребра принадлежат **разным** компонентам → добавляем ребро в MST, объединяем компоненты (`union_sets`)
   - Если концы в **одном** компоненте → пропускаем (добавление создало бы цикл)
4. Останавливаемся, когда MST содержит `n-1` ребро

```
Граф:        Отсортированные рёбра:    MST:
1 - 2: 4     1-3: 1 ✓ union(1,3)       1
1 - 3: 1     1-2: 4 ✓ union(1,2)      / \
2 - 3: 2     2-3: 2 ✗ (1,2,3 уже      3   2
2 - 4: 3             в одном           |
3 - 4: 5             компоненте)       4  <- через 2-4: 3 ✓
```

## Задача 3 (хард): DSU через деревья с отложенной оптимизацией

Наивная реализация `union` без эвристик может привести к вырожденному дереву высоты O(n). Цель этой задачи — реализовать DSU так, чтобы все операции работали за O(log\*n), где log\* — итерированный логарифм (практически константа для любых реальных данных).

### Требования

Реализовать DSU с двумя эвристиками одновременно:

1. **Объединение по рангу (union by rank)** — всегда присоединяем дерево меньшего ранга к дереву большего ранга.
2. **Сжатие пути (path compression)** — при каждом `find_set` перевешиваем все узлы на пути к корню напрямую к корню.

```c
typedef struct DSNode {
    long value;
    struct DSNode *parent;
    int rank;  // добавляется в хард-версии
} DSNode;
```

### Анализ сложности

Покажите (в отчёте или в тестах), что:
- Наивная реализация на цепочке N элементов даёт глубину O(n)
- Реализация с обеими эвристиками — O(log\*n) амортизированно
